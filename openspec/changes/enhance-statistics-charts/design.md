## Context

统计页当前基于本地 Room 账单记录展示月度统计，包括分类占比、分类排行、每日对比和月度对比。本次增强重点是提升图表可读性和交互反馈：分类占比需要标注每个扇区含义，每日/月度柱状图需要可点击选中并展示金额，本月排行需要按账单金额展示消费记录。

## Goals / Non-Goals

**Goals:**

- 在不新增网络依赖和不改变 Room 数据模型的前提下增强统计页展示。
- 分类占比图支持扇区引导线和标签，并尽量避免标签重叠。
- 每日对比和月度对比支持选中态、未选中态和选中金额提示。
- 新增当前月份消费排行列表，按消费金额从高到低排序。
- 保留现有统计页支出/入账切换、年月筛选和分类详情入口。

**Non-Goals:**

- 不新增服务端同步、账号体系或云备份。
- 不修改账单数据库表结构。
- 不替换现有 Compose 技术栈。
- 不改变明细页、新增记账页或分类详情页的核心业务流程。

## Decisions

- 使用现有统计数据在 UI 层计算图表展示项。
  - Rationale: 当前统计页已持有选中月份记录和近几个月记录，本次只增强展示和交互，不需要新增 DAO 或数据库字段。
  - Alternative: 新增数据库聚合查询。当前数据规模和页面需求可以由内存聚合满足，新增查询会增加实现复杂度。

- 分类占比图继续使用 Compose Canvas 绘制。
  - Rationale: 现有圆环图若已基于 Canvas 实现，可以直接扩展引导线、标签和碰撞规避逻辑，避免引入第三方图表库。
  - Alternative: 引入图表库。会增加依赖和样式适配成本，不符合当前轻量实现。

- 分类标签采用左右分栏和最小垂直间距策略减少重叠。
  - Rationale: 扇区较多或占比接近时，标签直接放在扇区外侧容易重叠。按扇区中点角度分配左右侧，再按 y 坐标排序并拉开最小间距，能覆盖常见手机屏幕。
  - Alternative: 只显示 Top N 标签。会丢失用户要求的“每一块代表什么”的信息。

- 每日/月度柱状图选中态保存在页面本地 UI 状态。
  - Rationale: 选中柱子只是展示交互，不需要进入 ViewModel 或持久化；月份切换或模式切换后可重置到当前默认选中项。
  - Alternative: 写入全局 UI state。会让 ViewModel 承担纯展示状态，增加状态同步成本。

- 选中提示浮层在图表区域内做边界钳制。
  - Rationale: 柱子靠近屏幕左右边缘时，提示浮层直接居中会超出屏幕；通过最大/最小 x 坐标限制保证完整可见。
  - Alternative: 统一固定到图表顶部中间。实现简单但无法直观关联选中柱子。

- 当前月份排行列表按单条账单金额降序展示。
  - Rationale: 用户给出的参考图是“本月支出排行”记录列表，而不是分类汇总排行；因此应按当前模式下的单条账单金额从高到低排序。
  - Alternative: 按分类汇总排行。已有分类排行能力，不能满足“这个月的排行”参考图中的逐笔记录展示。

## Risks / Trade-offs

- [Risk] 分类过多时所有标签都显示可能拥挤 → 使用最小间距、左右分栏和可读性优先策略；极小占比可用短标签但仍保留图例列表。
- [Risk] Canvas 点击命中区域过窄 → 给柱状图每个柱子保留稳定触摸宽度，窄柱也使用扩展命中区域。
- [Risk] 提示浮层遮挡图表 → 控制浮层高度并贴近选中柱顶部，必要时固定在图表内部上方。
- [Risk] 金额过大导致坐标或标签显示拥挤 → 使用金额格式化和纵轴分档，避免长文本撑破布局。
- [Risk] 小屏幕统计页内容变长 → 统计页应保持可滚动，排行列表不应导致页面底部内容不可达。
